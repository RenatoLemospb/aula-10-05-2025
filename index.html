Como Enviar Atualizações Locais para a Branch Main do GitHub

Publicado em: 12 de Maio de 2025

Este guia passo a passo mostra como enviar suas alterações locais para a branch main de um repositório no GitHub. Certifique-se de que você já clonou o repositório e configurou o Git localmente.

Passo a Passo

  1. Verifique o status do repositório:

    Confira quais arquivos foram modificados ou adicionados.

    git status
  2. Adicione as alterações:

    Adicione os arquivos modificados ao staging para prepará-los para o commit. 

    git add .

    (Use git add nome-do-arquivo para adicionar arquivos específicos.)

  3. Crie um commit:

    Registre suas alterações com uma mensagem descritiva.

    git commit -m "Descrição das alterações"
  4. Verifique a branch atual:

    Certifique-se de que você está na branch main.

    git branch

    Se necessário, mude para a branch main com:

    git checkout main
  5. Puxe as atualizações remotas:

    Antes de enviar, synchronize seu repositório local com o remoto para evitar conflitos. 

    git pull origin main
  6. Envie as alterações para o GitHub:

    Envie seus commits locais para a branch main no repositório remoto. 

    git push origin main

Nota: Se o repositório remoto estiver protegido, você pode precisar criar um pull request em vez de enviar diretamente para a main. Nesse caso, considere trabalhar em uma branch separada e abrir um PR no GitHub.
