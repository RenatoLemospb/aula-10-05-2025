Como Enviar Atualizações para uma Branch e Fazer Merge no GitHub

Publicado em: 15 de Maio de 2025

Este guia mostra como criar uma nova branch, enviar alterações locais para ela no GitHub e, em seguida, mesclar (merge) essas alterações na branch main usando um pull request.

Passo a Passo

  1. Crie uma nova branch:

    Crie uma branch chamada feature (ou outro nome de sua escolha).

    git checkout -b feature
  2. Verifique o status do repositório:

    Confira quais arquivos foram modificados ou adicionados.

    git status
  3. Adicione as alterações:

    Adicione os arquivos modificados ao staging.

    git add .

    (Use git add nome-do-arquivo para arquivos específicos.)

  4. Crie um commit:

    Registre suas alterações com uma mensagem descritiva.

    git commit -m "Adiciona nova funcionalidade"
  5. Envie a branch para o GitHub:

    Envie a branch feature para o repositório remoto.

    git push origin feature
  6. Crie um Pull Request (PR):

    Acesse o repositório no GitHub, vá para a aba "Pull Requests" e clique em "New Pull Request". Selecione feature como branch de origem e main como destino. Adicione uma descrição e envie o PR.

  7. Revise e faça o merge:

    Se necessário, peça a revisão de colegas. Após aprovação, clique em "Merge Pull Request" no GitHub e, em seguida, em "Confirm Merge". Opcionalmente, delete a branch feature no GitHub.

  8. Atualize o repositório local:

    Volte para a branch main localmente e puxe as alterações mescladas.

    git checkout main
    git pull origin main
  9. (Opcional) Delete a branch local:

    Se a branch feature não for mais necessária, delete-a localmente.

    git branch -d feature

Nota: Se houver conflitos durante o merge, o GitHub indicará. Você pode resolvê-los no GitHub ou localmente com git merge após puxar a branch main.

Como Enviar Atualizações Locais para a Branch Main do GitHub

Publicado em: 12 de Maio de 2025

Este guia passo a passo mostra como enviar suas alterações locais para a branch main de um repositório no GitHub. Certifique-se de que você já clonou o repositório e configurou o Git localmente.

Passo a Passo

  1. Verifique o status do repositório:
    git status
  2. Adicione as alterações:
    git add .
  3. Crie um commit:
    git commit -m "Descrição das alterações"
  4. Verifique a branch atual:
    git branch

    Se necessário, mude para main:

    git checkout main

  5. Puxe as atualizações remotas:
    git pull origin main
  6. Envie as alterações para o GitHub:
    git push origin main

Nota: Se o repositório remoto estiver protegido, você pode precisar criar um pull request.
