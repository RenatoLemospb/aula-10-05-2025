Base de Conhecimento: Comandos Git

Publicado em: 18 de Maio de 2025

Esta é uma referência completa dos principais comandos Git para gerenciar repositórios, branches, commits, tags e mais. Use esta base de conhecimento para revisar os comandos essenciais.

Configuração

  • Definir nome do usuário:

    Configura o nome do autor dos commits.

    git config --global user.name "Fulano de Tal"
  • Definir e-mail do usuário:

    Configura o e-mail do autor dos commits.

    git config --global user.email fulanodetal@exemplo.br
  • Listar configurações:

    Exibe todas as configurações do Git.

    git config --list
  • Defin
  • Definir VSCode como editor padrão:

    Permite escrever mensagens de commit no VSCode.

    git config --global core.editor "code --wait"
  • Resetar configuração do editor:

    Remove a configuração do editor padrão.

    git config --global --unset core.editor

Repositório

  • Criar repositório local:

    Inicializa um novo repositório Git.

    git init
  • Vincular repositório remoto:

    Conecta o repositório local a um remoto.

    git remote add origin <url_do_repositório_remoto>
  • Atualizar branch local com remoto:

    Sincroniza a branch local com a remota.

    git pull origin <nome_branch>

Branches

  • Criar e entrar em uma nova branch:

    Cria uma branch e alterna para ela.

    git checkout -b <nome_branch>
  • Renomear branch:

    Altera o nome da branch atual.

    git branch -m <novo_nome_branch>
  • Excluir branch:

    Remove uma branch local.

    git branch -d <nome_branch>

Commits

  • Adicionar arquivos ao staging:

    Prepara arquivos para o commit (use . para todos).

    git add <nome_arquivo>
  • Comentar alterações:

    Registra alterações no repositório com uma mensagem.

    git commit -m "Sua mensagem de commit aqui"
  • Editar mensagem do commit:

    Permite alterar a mensagem do último commit antes do push.

    git commit --amend
  • Enviar alterações para branch remota:

    Envia commits locais para o repositório remoto.

    git push origin <nome_branch>
  • Anular commit (manter arquivos no staging):

    Desfaz o último commit, mantendo alterações no staging.

    git reset --soft HEAD~1
  • Anular commit (voltar ao working directory):

    Desfaz o commit e remove do staging.

    git reset --mixed HEAD~1
  • Anular commit e apagar alterações:

    Desfaz o commit e descarta todas as alterações.

    git reset --hard HEAD~1
  • Reverter commit:

    Cria um novo commit que desfaz o anterior.

    git revert HEAD~1

Stash

  • Salvar alterações no stash:

    Guarda alterações não commitadas para usar em outra branch.

    git stash save "mensagem"
  • Listar stashes:

    Exibe todas as alterações salvas no stash.

    git stash list
  • Recuperar stash:

    Restaura as alterações do stash na posição 0.

    git stash pop 0
  • Limpar stash:

    Remove todas as alterações salvas no stash.

    git stash clear

Histórico

  • Ver histórico de commits:

    Exibe o log completo de commits.

    git log
  • Histórico de uma pasta:

    Mostra o log de alterações em uma pasta específica.

    git log <nome_pasta>
  • Histórico de um arquivo:

    Mostra o log de alterações de um arquivo específico.

    git log <nome_arquivo>
  • Histórico resumido:

    Exibe o log de forma compacta.

    git log --oneline
  • Histórico com ramificação:

    Mostra o log com gráfico de branches.

    git log --graph
  • Histórico gráfico (interface):

    Abre uma interface gráfica para visualizar o log.

    gitk

Tags

  • Listar tags:

    Exibe todas as tags do repositório.

    git tag
  • Listar tags com filtro:

    Filtra tags com base em um padrão.

    git tag -l "v1.8.5*"
  • Visualizar conteúdo de uma tag:

    Mostra detalhes de uma tag específica.

    git show v1.4
  • Criar tag anotada:

    Cria uma tag com mensagem e metadados.

    git tag -a v1.4 -m "my version 1.4"
  • Criar tag leve:

    Cria uma tag simples, sem metadados.

    git tag v1.4
    • Criar tag para commit anterior:

    Associa uma tag a um commit específico (use o hash do commit).

    git tag -a v1.2 9fceb02

    Para encontrar o hash, use:

    git log --pretty=oneline

  • Enviar tag para o remoto:

    Envia uma tag específica para o repositório remoto.

    git push origin v1.4
  • Enviar todas as tags:

    Envia todas as tags pendentes para o remoto.

    git push origin --tags
  • Excluir tag:

    Remove uma tag local (use o padrão para múltiplas tags).

    git tag -d $(git tag -l "tag_prefix")

Como Enviar Atualizações para uma Branch e Fazer Merge no GitHub

Publicado em: 15 de Maio de 2025

Este guia mostra como criar uma nova branch, enviar alterações locais para ela no GitHub e, em seguida, mesclar (merge) essas alterações na branch main usando um pull request.

Passo a Passo

  1. Crie uma nova branch:
    git checkout -b feature
  2. Verifique o status do repositório:
    git status
  3. Adicione as alterações:
    git add .
  4. Crie um commit:
    git commit -m "Adiciona nova funcionalidade"
  5. Envie a branch para o GitHub:
    git push origin feature
  6. Crie um Pull Request (PR):

    Acesse o repositório no GitHub, vá para a aba "Pull Requests" e clique em "New Pull Request".

  7. Revise e faça o merge:

    Após aprovação, clique em "Merge Pull Request" no GitHub.

  8. Atualize o repositório local:
    git checkout main
    git pull origin main
  9. (Opcional) Delete a branch local:
    git branch -d feature

Nota: Resolva conflitos no GitHub ou localmente, se necessário.

Como Enviar Atualizações Locais para a Branch Main do GitHub

Publicado em: 12 de Maio de 2025

Este guia passo a passo mostra como enviar suas alterações locais para a branch main de um repositório no GitHub.

Passo a Passo

  1. Verifique o status do repositório:
    git status
  2. Adicione as alterações:
    git add .
  3. Crie um commit:
    git commit -m "Descrição das alterações"
  4. Verifique a branch atual:
    git branch

    Se necessário, mude para main:

    git checkout main

  5. Puxe as atualizações remotas:
    git pull origin main
  6. Envie as alterações para o GitHub:
    git push origin main

Nota: Se o repositório remoto estiver protegido, crie um pull request.
